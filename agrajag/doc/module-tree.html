Module Hierarchy

  • application: Main Agrajag application class with helper classes.
  • background: Plain and simple space-style background.
  • base: This module contains base definitions needed by different modules.
  • clock: Thin wrapper for pygame.Clock.
  • dbmanager: Management of XML data describing game objects (enemies, bonuses, projectiles, etc.).
  • functions: Helper functions for normalization, conversion, etc.
  • gfxmanager: Management and conversion to on-screen colour pallette of graphics.
  • groupmanager: Management of sprite groups.
  • hud: In-level heads-up display.
  • mover: Definitions of different types of movers (objects thay control movement of other objects).
  • obstacle: Dummy object intended to place on a level just to display something.
  • run
  • signals: File: signals.py Author: Patrick Chasco Created: July 26, 2005
  • spaceship: Enemies, player ship, gear, projectiles, bonuses and maybe some other things.
  • stagemanager: Management of XML defined levels.
  • widgets: Displayables for the HUD.
  • xmlmanager: Backend for XML processing managers.
